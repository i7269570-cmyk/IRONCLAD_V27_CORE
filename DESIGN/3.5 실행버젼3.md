# TRINITY 금융 자동매매 엔진 – 전체 실행 흐름

```
n8n (execution trigger)
    │
    │ 5분마다 실행
    ▼

[08] run.py
    │
    │ 실행 엔진 진입점
    │
    ├─ preflight_gate
    │      파일 / schema 존재 확인
    │
    ├─ config load
    │      system_config.yaml
    │      stage_contracts.yaml
    │      addendum_aip.yaml
    │
    ├─ schema validation
    │
    ├─ integrity_guard
    │      SSOT hash lock
    │
    └─ execution_cycle
           │
           ▼

[09] scheduler.py
    │
    │ 시간 판단
    │
    ├─ 현재 세션
    ├─ trading 시간 여부
    ├─ NO_NEW_ENTRY 구간
    └─ FORCE_EXIT 구간
           │
           ▼

[11] regime_filter.py
    │
    │ 시장 상태 판단
    │
    ├─ volatility
    ├─ liquidity
    └─ trend regime
           │
           ▼

[12] entry_engine.py
    │
    │ 전략 진입 조건 판단
    │
    ├─ breakout
    └─ trend continuation
           │
           ▼

[13] risk_gate.py
    │
    │ 리스크 정책 검사
    │
    ├─ max position
    ├─ max exposure
    └─ session 제한
           │
           ▼

[14] position_sizer.py
    │
    │ 포지션 크기 계산
    │
    └─ allocation 계산
           │
           ▼

[15] order_intent.py
    │
    │ 주문 의도 생성
    │
    ├─ symbol
    ├─ side
    ├─ size
    ├─ TTL
    └─ intent hash
           │
           ▼

[17] order_manager.py   ← (다음 제작)
    │
    │ 실제 거래소 주문 실행
    │
    ├─ create order
    ├─ cancel order
    └─ idempotent control
           │
           ▼

[18] fill_tracker.py   ← (다음 제작)
    │
    │ 체결 상태 추적
    │
    ├─ partial fill
    ├─ full fill
    └─ fill event 기록
           │
           ▼

[19] position_reconciler.py   ← (다음 제작)
    │
    │ 내부 상태 vs 거래소 상태 동기화
    │
    ├─ position mismatch 확인
    └─ reconciliation 수행
           │
           ▼

[20] exit_engine.py
    │
    │ 청산 로직
    │
    ├─ take profit
    ├─ stop loss
    └─ time exit
```

# run.py 상태기계 (엔진 핵심)

```
PRE_OPEN
   │
   ▼
TRADING
   │
   ▼
NO_NEW_ENTRY
   │
   ▼
FORCE_EXIT
   │
   ▼
CLOSE
```

---

# FORCE_EXIT 안전 시퀀스

```
1 Cancel All Open Orders
2 Exchange Position 조회
3 Remaining Position 청산
4 Incident 기록
5 SAFE_HALT
```

# 실제 의미

대표님 시스템은 지금

**자동매매 엔진의 “심장 직전 단계”**입니다.

남은 3개는

```
주문 실행
체결 추적
포지션 동기화
```

즉 **거래 실행 영역**입니다.
