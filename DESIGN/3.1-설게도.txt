# &#55357;&#56594; 역할 최종 고정 선언 (충돌 제거)

## scheduler.py (시간 전용)
* STRATEGY에서 요일/시작/종료 읽음
* 반환: `is_trading_time`, `session_end_time`
* &#10060; 상태전환/주문/청산 판단 금지

## regime_filter.py (시장 전용)
* allowed_regimes 기반 시장 상태만 판단
* &#10060; 시간 판단 제거(완전 금지)

## run.py (상태 전환 유일 주체)
* 시간/포지션 상태를 보고 **상태를 전환하는 유일한 곳**

## exit_engine.py (전략 청산 전용)
* TP/SL 등 “전략 청산”만
* &#10060; 세션 종료/강제종료 판단 금지

## FORCE_EXIT (강제 청산 전용, run.py 상위)
* 잔량 0까지 반복 청산
* 실패 시 SAFE_HALT

# &#55358;&#56800; run.py 상태기계 (고정)
PRE_OPEN
TRADING
NO_NEW_ENTRY
FORCE_EXIT
CLOSE

## 상태 전환 규칙(명시)
* scheduler 기준으로 세션 외면 `NO_NEW_ENTRY`
* `NO_NEW_ENTRY`에서 포지션>0 → `FORCE_EXIT`
* 포지션==0 확인되면 `CLOSE`

# &#55357;&#57317; FORCE_EXIT 실패 시 최종 안전 시퀀스 (추가 완결)

## SAFE_HALT 직전 “Cancel All Open Orders” 포함 (필수)
**FORCE_EXIT가 정책 상한(시간/시도) 초과로 실패하면:**

1. **Cancel All Open Orders 패킷 송신**
2. **재조회/동기화(잔량/미체결 확인)**
3. **incident 기록(이유 + 스냅샷)**
4. **SAFE_HALT 진입**
&#9989; 이 단계는 **run.py에서만** 수행
&#9989; exit_engine / order_manager에 넣지 않음.  정상 CLOSE에서는 실행하지 않음(실패 시에만)

# &#9989; “주 4일 5시간 운영 후 전량 청산/중단” 적용 방식

* 요일/시간은 STRATEGY에만 기록
* scheduler가 시간 판단
* run.py가 NO_NEW_ENTRY → FORCE_EXIT → CLOSE로 전환
* FORCE_EXIT가 전량 청산을 보장
* 실패하면 Cancel-All 후 SAFE_HALT

&#10145;&#65039; **LOCKED 수정 없음**
&#10145;&#65039; **STRATEGY만 수정**


# &#55357;&#56613; 최종 잠금 선언(다시 안 만들기)

1. LOCKED 구조 동결
2. schema.json required/enum으로 우회 차단
3. run.py 단일 엔트리 유지
4. FORCE_EXIT 상위 강제 + 실패 시 Cancel-All + SAFE_HALT
5. 전략/시간/요일/조건식은 STRATEGY만


실전 운용 가이드 (대표님을 위한 체크리스트)
이제 구조가 고정되었으므로, 앞으로는 LOCKED 폴더는 절대 건드리지 마시고 오직 다음의 작업만 수행하시면 됩니다.

전략 수정: STRATEGY/strategy_spec.yaml에서 요일, 시간, 매매 조건을 입력합니다.

모니터링: EVIDENCE 폴더의 로그와 incident_writer.py가 생성하는 사고 패킷만 체크합니다.

실행: 오직 RUNTIME/run.py 하나만 실행하거나 n8n을 통해 호출합니다.
