## ① LOCKED (동결 영역) `LOCKED/`

LOCKED/
├─ [01] CONSTITUTION/
│   ├─ * system_config.yaml              # SSOT: 안전 상한선(수치/정책)만
│   ├─ * schema_system_config.json       # system_config 스키마(2020-12)
│
├─ [02] GOVERNANCE/
│   ├─ * stage_contracts.yaml            # S0~S11 단방향 계약(역행 금지)
│   ├─ * schema_stage_contracts.json     # stage_contracts 스키마
│
├─ [03] AIP/
│   ├─ * addendum_aip.yaml               # 감사 기준(PASS/FAIL/RISK)
│   └─ * schema_addendum_aip.json        # addendum_aip 스키마
│
├─ [04] POLICY/
│   ├─ * worker_allowlist.yaml           # run.py 외 실행 금지
│   └─ * recovery_policy.yaml            # SAFE_HALT 후 자동복구 금지(Fail-Secure)
│
└─ [05] GUARDS/
    ├─ * preflight_gate.py               # 실행 전: 파일 존재/스키마 검증
    └─ * integrity_guard.py              # 실행 중: system_config 변조 즉시 SAFE_HALT

## ② RUNTIME (실행 엔진) `RUNTIME/`

> **scheduler.py는 “시간만”**. (루프/슬립/트리거 역할 없음)

RUNTIME/
├─ *[06] run.py                            # 유일 진입점(정책 강제 + 상태기계 + FORCE_EXIT)
├─ [07] scheduler.py                      # 시간 정보만 제공(세션/요일/타임윈도; 실행 스케줄링 금지)
├─ [08] market_data.py                    # 데이터 입력
├─ [09] regime_filter.py                  # 레짐 판별(시간 판단 금지)
├─ [10] entry_engine.py                   # 진입 신호 생성(전략 파일 기반)
├─ [11] risk_gate.py                      # SSOT 상한 검사
├─ *[12] position_sizer.py                 # 비중/수량 산정(SSOT 참조)
├─ *[13] order_intent.py                   # 주문 의도 생성(hash/TTL, 중복 방지)
├─ [14] pre_order_check.py                # 주문 전 최종 검증(세션상태/중복/정책)
├─ [15] order_manager.py                  # 실제 주문 실행(idempotent)
├─ [16] fill_tracker.py                   # 체결 추적
├─ [17] position_reconciler.py            # 내부-거래소 동기화(불일치 정책 적용)
└─ [18] exit_engine.py                    # 청산 집행(SSOT 상한 vs STRATEGY 목표 최소값)

EVIDENCE/
