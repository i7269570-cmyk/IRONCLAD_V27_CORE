# 🔥 핵심 실행 로직 (설명 포함 완성)

---

## 1️⃣ scheduler.py (시간 로직)

```python
if time < 09:00:
    mode = "CLOSED"
    # 장 시작 전 → 완전 정지 상태

elif 09:00 <= time < 14:50:
    mode = "TRADE"
    # 정상 매매 구간
    # → 진입 + 청산 모두 가능

elif 14:50 <= time < 15:00:
    mode = "NO_ENTRY"
    # 신규 진입 금지 구간
    # → 기존 포지션 정리 준비 단계
    # → 청산만 허용

elif 15:00 <= time < 15:30:
    mode = "FORCE_EXIT"
    # 강제 청산 구간
    # → 모든 포지션 반드시 정리
    # → 15:30 전에 0 포지션 만들어야 함

else:
    mode = "CLOSED"
    # 15:30 이후
    # → 이미 포지션 0 상태
    # → 아무것도 하지 않음
```

---

## 2️⃣ run.py 흐름 (상태기계)

```text
scheduler → mode 확인

TRADE:
→ 진입 + 청산 모두 실행

NO_ENTRY:
→ 진입 금지
→ 청산만 수행

FORCE_EXIT:
→ 모든 포지션 강제 청산
→ 신규 진입 절대 금지

CLOSED:
→ 완전 정지 (이미 현금 상태)
```

---

## 3️⃣ exit_engine.py (청산 핵심)

```python
if mode == "FORCE_EXIT":
    close_all_positions()
    # 15:00 이후 강제 청산 시작
    # 15:30 전에 반드시 포지션 0 만들어야 함

elif mode == "NO_ENTRY":
    manage_existing_positions()
    # 신규 진입 금지 상태에서 기존 포지션만 정리

else:
    apply_tp_sl()
    # 일반 구간에서는 손절/익절만 수행
```

## 4️⃣ risk_gate.py (position sizing 포함)

```python
if current_positions >= 2:
    reject_trade()
    # 최대 2포지션 제한

if same_asset_exists:
    reject_trade()
    # 동일 자산군 중복 금지

# position sizing 계산
if asset == "STOCK":
    size = 0.5
    # 주식은 상대적으로 안정 → 큰 비중

elif asset == "CRYPTO":
    size = 0.3
    # 코인은 변동성 큼 → 작은 비중

if total_exposure + size > 1.0:
    reject_trade()
    # 총 자본 100% 초과 금지

return size
# 최종 주문 크기 반환 → order_manager로 전달
```
