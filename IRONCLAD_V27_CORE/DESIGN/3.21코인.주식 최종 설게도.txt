                               risk_gate.py에서 계산 후 통과/차단 결정 → order_manager로 전달.

│  ├─ order_manager.py                      # 주문 실행·재시도·중복 주문 방지(Idempotent)
│  ├─ fill_tracker.py                       # 체결 추적 및 체결 상태 반영

      position_sizer.py                        얼마 넣을지

│  ├─ strategy_spec.yaml                    # 요일·시간·진입/청산·포지션 관련 전략 파라미터
│  ├─ selector_rules.yaml                   # 스캔 축소 규칙 (거래량/거래대금 기준 Top-N)
│  ├─ entry_rules.yaml                      # 조건식 정의

   ├─ ledger/                               # Append-only 매매 원장

   이 구조의 최종 평가
장점:
- 과한 폴더 구조 제거
- Top-N 스캔으로 지연 감소
- atomic_write 반영 가능
- SAFE_HALT 철학 유지
- 실전 운영에 맞는 최소구조
주의:
- 최대 2포지션이면 state.json 구조를 리스트 기반으로 설계해야 함
- 동일 자산군 중복 금지는 risk_gate에서 강제해야 함
- 15:30 이후 신규 진입 금지 + 청산 모드는 scheduler와 exit_engine이 함께 처리해야 함
