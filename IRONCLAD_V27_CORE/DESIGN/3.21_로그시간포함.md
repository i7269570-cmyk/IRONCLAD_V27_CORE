# 🏗️ IRONCLAD_V30.1_FINAL — 🔒 FROZEN (최종 고정 설계도)

```text
IRONCLAD_V30.1_FINAL/

├─ LOCKED/                                      # 절대 규칙 (수정 금지)
│  ├─ system_config.yaml                        # 리스크 상한 (2포지션, 손실 한도, FORCE_EXIT)
│  ├─ schema.json                               # 구조 검증
│  ├─ recovery_policy.yaml                      # SAFE_HALT 후 자동 복구 금지
│  └─ GUARDS/
│     ├─ preflight_gate.py                      # 실행 전 검증
│     └─ integrity_guard.py                     # 실행 중 변경 감지

├─ STRATEGY/                                    # 전략 정의 (여기만 수정 가능)
│  ├─ strategy_spec.yaml                        # 시간/포지션/전략 파라미터
│  ├─ selector_rules.yaml                       # Top-N (30~50)
│  ├─ entry_rules.yaml                          # 진입 조건
│  └─ exit_rules.yaml                           # 청산 조건

├─ RUNTIME/                                     # 실행 엔진 (코드 구현 영역)
│  ├─ run.py                                    # 상태기계 (유일 진입점)
│  ├─ scheduler.py                              # 시간 모드 결정
│  ├─ data_loader.py                            # Top-N 데이터 로드
│  ├─ selector.py                               # 30~50 → 3~5 선택
│  ├─ regime_filter.py                          # 시장 상태 판단
│  ├─ entry_engine.py                           # 진입 판단
│  ├─ risk_gate.py                              # 리스크 + position sizing
│  ├─ pre_order_check.py                        # 주문 전 최종 검증
│  ├─ order_manager.py                          # 주문 실행
│  ├─ fill_tracker.py                           # 체결 추적
│  ├─ position_reconciler.py                    # 상태 복구
│  ├─ state_manager.py                          # atomic write
│  ├─ ledger_writer.py                          # 거래 로그 기록
│  ├─ exit_engine.py                            # 청산 로직
│  └─ exception_handler.py                      # Cancel-All → SAFE_HALT

├─ STATE/
│  ├─ state.json                                # 포지션 상태
│  └─ runtime_status.json                       # 실행 상태

└─ EVIDENCE/
   ├─ ledger/                                   # 거래 로그 파일 저장
   └─ incident/                                 # 사고 기록
```

# 🔥 핵심 실행 로직 (고정)

```text
run.py

1. preflight_gate
2. scheduler (시간 판단)

3. data_loader (Top-N만)
4. selector (3~5개)

5. regime_filter
6. entry_engine

7. risk_gate (position sizing 포함)

8. pre_order_check
9. order_manager

10. fill_tracker
11. state_manager (atomic write)
12. ledger_writer (로그 기록)

13. exit_engine
14. position_reconciler

15. 예외 발생:
   Cancel-All → incident 기록 → SAFE_HALT
```

# 🔥 시간 구조 (절대 고정)

```text
09:00 ~ 14:50   → TRADE (진입 + 청산)

14:50 ~ 15:00   → NO_ENTRY (진입 금지, 청산만)

15:00 ~ 15:30   → FORCE_EXIT (전량 청산 구간)

15:30 이후      → CLOSED (이미 현금 상태)

```
