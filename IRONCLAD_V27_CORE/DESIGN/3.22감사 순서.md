**파일 만드는 순서 + 각 단계 후 감사 내용**

1. run.py  
   → 넣을 것: 전체 흐름 호출 순서 + scheduler/risk_gate 호출  
   → 감사: run.py 단독 + scheduler.py + risk_gate.py 3개 부분 감사 (PASS 받으면 보관)

2. scheduler.py  
   → 넣을 것: 시간대별 모드 결정 로직 (TRADE/NO_ENTRY/FORCE_EXIT/CLOSED)  
   → 감사: 위 3개 함께 (run + scheduler + risk_gate)

3. risk_gate.py  
   → 넣을 것: 최대 2포지션, 동일 자산군 금지, 노출 한도, position sizing 계산  
   → 감사: 위 3개 부분 감사 완료

4. preflight_gate.py + integrity_guard.py  
   → 넣을 것: 실행 전 검증 + 실행 중 감시  
   → 감사: GUARDS 2개 + run.py (4개 파일 감사)

5. state_manager.py  
   → 넣을 것: atomic write (temp → fsync → replace)  
   → 감사: state 관련 5개 파일 (run/scheduler/risk_gate/preflight/integrity/state)

6. exception_handler.py  
   → 넣을 것: Cancel-All → incident 기록 → SAFE_HALT  
   → 감사: 6개 파일 (위 + exception)

7. data_loader.py → selector.py → regime_filter.py → entry_engine.py  
   → 넣을 것: Top-N만 로드 → 3~5개 선정 → 시장 필터 → 진입 조건 평가  
   → 감사: 7개 핵심 파일 전체 감사 (LOCKED/STRATEGY yaml 제외)

8. LOCKED/system_config.yaml + schema.json + recovery_policy.yaml  
   → 넣을 것: 리스크 상한선 + 스키마 + 복구 금지 정책  
   → 감사: 7개 파일 + LOCKED 3개 (10개)

9. STRATEGY/strategy_spec.yaml + selector_rules.yaml + entry_rules.yaml + exit_rules.yaml  
   → 넣을 것: 전략 파라미터 + Top-N 기준 + 진입/청산 조건식  
   → 감사: 최종 9개 + STRATEGY 4개 통합 감사 (완료)

10. 나머지 RUNTIME 파일들 (pre_order_check → order_manager → fill_tracker → ledger_writer → exit_engine → position_reconciler)  
    → 넣을 것: 각 모듈 역할 구현  
    → 감사: 전체 구조 최종 확인 (필요 시)

이 순서대로 하면 감사 단계가 명확해집니다.
